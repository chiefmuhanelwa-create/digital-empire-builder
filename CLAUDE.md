## This Project ‚Äî Technical Reference

### Commands

```bash
bun install          # Install dependencies (use BUN, not npm or yarn)
bun dev              # Local dev server (port 3000)
bun run build        # Production build for Cloudflare Workers
bunx wrangler deploy # Deploy to Cloudflare Workers (requires all env vars set)
bunx tsc --noEmit    # Type-check only ‚Äî run BEFORE and AFTER every code change
```

### Architecture

**Stack:** TanStack Start v1 (SSR) + React 19 + Vite 7 ‚Äî **NOT Next.js**
**Hosting:** Cloudflare Workers (`wrangler.jsonc` configured)
**Database:** Supabase PostgreSQL (14 tables, RLS on all, 22 migrations)
**Auth:** Supabase magic-link + password. First user auto-becomes admin via DB trigger.
**Payments:** Paystack (ZAR, SA/Africa) + Stripe (USD/GBP/EUR, international) ‚Äî dual-rail
**Email transactional:** Lovable Cloud via pgmq + pg_cron queues, 7 React-Email templates
**Email marketing:** MailerLite (sequences, segments, broadcasts)
**CAPTCHA:** Cloudflare Turnstile (required on all public forms)
**Package manager:** Bun (not npm, not yarn, not pnpm)

### Non-Negotiable Rules

1. **Bun only** ‚Äî `bun add`, `bun remove`, `bunx`. Never `npm install` or `yarn add`.
2. **TanStack, not Next.js** ‚Äî No `app/` directory. No `use server`. No `getServerSideProps`. No `useRouter` from `next/navigation`.
3. **Always type-check** ‚Äî `bunx tsc --noEmit` before AND after every code change.
4. **HMAC on all webhooks** ‚Äî Both Paystack and Stripe webhooks must verify signatures before processing.
5. **Turnstile everywhere public** ‚Äî `/apply`, `/signup`, `/login`, `/contact`, checkout init all require Cloudflare Turnstile validation.
6. **Never edit migrations** ‚Äî `supabase/migrations/` files are immutable. Always create a new `.sql` file.
7. **SARS 25% reserve** ‚Äî Every payment must store `tax_reserve_cents = ROUND(total_cents * 0.25)`. This field is not yet in schema ‚Äî add via new migration.
8. **No Anthropic API here** ‚Äî AI generation lives in `full-content-system`. This project delivers products, not generates content.
9. **No hardcoded prices** ‚Äî Always read `price_cents` from DB. Never write `18000 * 100` in code.
10. **Proof numbers only** ‚Äî Never fabricate story outcomes. Use the verified table above.

### Payment Architecture ‚Äî Dual Rail

```
ZAR (SA + Africa)          ‚Üí Paystack
  Secret: PAYSTACK_SECRET_KEY
  Public: VITE_PAYSTACK_PUBLIC_KEY
  Webhook: /api/public/paystack-webhook (verify x-paystack-signature HMAC)

USD/GBP/EUR/AUD (global)   ‚Üí Stripe
  Secret: STRIPE_SECRET_KEY
  Public: VITE_STRIPE_PUBLISHABLE_KEY
  Webhook: /api/public/stripe-webhook (verify stripe-signature, STRIPE_WEBHOOK_SECRET)
  Orders: provider = 'stripe'

Currency routing:
  Cloudflare CF-IPCountry header ‚Üí ZA/NG/GH/KE/UG/TZ ‚Üí Paystack
  All other countries ‚Üí Stripe
  User can override with manual currency picker on checkout
```
